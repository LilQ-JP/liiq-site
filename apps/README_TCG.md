# TCGオンライン対戦ツール (Web App) 開発・運用ガイド

## 概要
このディレクトリ（`public/tcg/`）には、P2P（PeerJS）を利用したサーバーレスのTCGオンライン対戦ツールの本体が格納されています。
Next.jsの `/public` ディレクトリに静的ファイルとして配置されているため、サイトと同じドメイン（例: `https://lilq.jp/tcg/`）で自動的にアクセス可能となります。

## 本番環境へのアクセス
- URL: `https://lilq.jp/tcg/`  
※ `public/tcg/index.html` が表示されます。

---

## 開発と動作確認の手順（エンジニア向け）

### 1. ローカルでの確認方法
Next.jsの開発サーバーが立ち上がっていれば、自動的にローカルホスト上で確認できます。

```bash
# プロジェクトルートで開発サーバーを起動
npm run dev
```

ブラウザで以下にアクセスしてください：
👉 `http://localhost:3000/tcg/`

### 2. アプリ本体（HTML）のコード更新方法
本アプリは単一のHTMLファイル（`index.html`）にCSS・JSがまとめられた形で提供されています。  
ローカルでコードを修正し、最新版を反映させるための手順は以下の通りです。

1. **修正の反映**
   新しいバージョンのHTMLファイルが作成・提供された場合、`public/tcg/index.html` を上書き保存（置換）してください。
   ```bash
   # 例: 新しいファイルが ~/Downloads/new_version.html の場合
   cp ~/Downloads/new_version.html public/tcg/index.html
   ```
2. **キャッシュに注意**
   ブラウザでキャッシュが強く効いている場合があるため、動作確認時は「スーパーリロード（Shift + F5 または Cmd + Shift + R）」を行うか、シークレットウィンドウをご利用ください。

3. **デプロイ**
   ファイルを上書きしたら、通常通り `git commit` および `push` を行うだけでGitHub Actionsによって再デプロイが行われ、本番の `https://lilq.jp/tcg/` に最新版が反映されます。

### 3. 注意事項
- **アセットパス（画像など）**
  もし今後アプリ内で画像を使う場合は、`public/tcg/assets/` などのディレクトリを作成し、HTMLからのURLパス指定を相対パス（`./assets/image.png` 等）で行うと安全です。
- **PeerJSについて**
  バックエンドサーバー（シグナリングサーバー含む）は外部の無料クラウド（0.peerjs.com）を利用しているためサーバー管理は不要ですが、一時的な接続障害が起こる可能性があります。
